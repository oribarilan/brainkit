# Architecture

## System Overview

Brainkit has two distinct layers that never overlap:

```
┌─────────────────────────────────────┐
│           brainkit CLI              │  ← Human-facing. Installs skills,
│  (npm package, TypeScript)          │     maintains config, generates
│                                     │     agent instructions.
│  • TUI for feature selection        │
│  • Writes brainkit.toml             │
│  • Installs/removes skill files     │     Never touches vault content.
│  • Generates AGENTS.md              │
│  • Multi-provider config generation │
│  • Version check / update awareness │
└─────────────┬───────────────────────┘
              │ installs
              ▼
┌─────────────────────────────────────┐
│          Agent Skills               │  ← Agent-facing. Installed into
│  (SKILL.md files per Agent Skills   │     the vault's skill directories.
│   specification)                    │
│                                     │
│  • /doctor — vault health, setup &   │     Agents read and execute these.
│             maintenance              │     Brainkit never executes them.
│  • /config — configuration mgmt     │
│  • /add-brag — bragfile management  │
│  • /process-notes — note processing │
│  • /query-contacts — people lookup  │
│  • /review — self-review drafting   │
│  • ...more over time                │
└─────────────────────────────────────┘
```

### Separation of Concerns

| Concern | Handled by | How |
|---|---|---|
| Feature selection | brainkit CLI | TUI with toggles |
| Skill installation | brainkit CLI | Copies SKILL.md files to provider dirs |
| Config management | brainkit CLI | Reads/writes brainkit.toml |
| Agent instructions | brainkit CLI | Generates AGENTS.md from config + features |
| Vault structure setup | Agent via `/doctor` skill | Creates dirs, files based on config |
| Vault migration | Agent via `/doctor` skill | Moves existing content safely |
| Configuration changes | Agent via `/config` skill | Edits brainkit.toml + patches AGENTS.md |
| Daily operations | Agent via skills | /add-brag, /process-notes, etc. |
| Vault health | Agent via `/doctor` skill | Validates structure, conventions |
| Vault cleanup | Agent via `/doctor` skill | Archives disabled feature content |

## File Layout (What Brainkit Creates)

Brainkit only creates/manages files in specific locations. Everything else in the vault is the agent's and user's domain.

```
my-vault/
  brainkit.toml                     # Brainkit config (managed by CLI)
  AGENTS.md                         # Agent instructions (generated by CLI)
  .gitignore                        # Default ignores (created by CLI)
  .agents/                          # Agent skills directory (managed by CLI)
    skills/
      brainkit/                     # Brainkit skill bundle
        SKILL.md                    # Core vault skill
      brainkit-config/
        SKILL.md                    # /config command
      brainkit-add-brag/
        SKILL.md                    # /add-brag command (if bragfile enabled)
      brainkit-process-notes/
        SKILL.md                    # /process-notes command
      ...
  .claude/                          # Claude Code provider pointer (generated)
    skills/
      brainkit/
        SKILL.md
  # Other provider dirs as needed

  # === Everything below is managed by the AGENT, not brainkit ===
  01_projects/                      # Created by agent via /doctor
  02_areas/
    career/
      bragfile.md                   # Created by agent via /doctor (if bragfile enabled)
  03_resources/
    contacts.md                     # Created by agent via /doctor (if contacts enabled)
  04_archive/
```

## Config File: `brainkit.toml`

The single source of truth for the vault's brainkit configuration. Tracks which features are enabled, user personalization, and brainkit version info.

```toml
[brainkit]
version = "1.0.0"                   # brainkit version that last wrote this config

[user]
name = "Ori Bar-ilan"
role = "AI/ML Software Engineer"
expertise = ["AI/ML", "security", "distributed-systems"]
tone = "direct and technical"
rules = []                          # User-defined rules (e.g. "Always use bullet points")
scope = "professional"              # "professional", "personal", or "both"
context = """
I use this brain to capture engineering decisions, \
meeting notes, and references. Prefer concise, actionable notes.
"""

[features]
bragfile = true
contacts = true
meeting-notes = true
self-review = true
vault-health = true

[agents]
providers = ["agents", "claude"]    # Which provider dirs to generate
```

### Config Discovery

When brainkit runs, it looks for `brainkit.toml` in the current directory. If found, it operates in "update" mode (show existing config, offer new features). If not found, it operates in "init" mode (full onboarding TUI).

## Multi-Provider Agent Config

Brainkit generates skill files for multiple AI agent providers. The primary target is the `.agents/` directory (the emerging cross-provider standard). Additional provider directories are generated as thin pointers or copies.

### Provider Strategy

| Provider | Directory | Strategy |
|---|---|---|
| Standard (.agents/) | `.agents/skills/` | Primary — full skill files |
| Claude Code | `.claude/skills/` | Thin pointer to AGENTS.md or full copy |
| Cursor | `.cursor/skills/` | Thin pointer to AGENTS.md or full copy |
| VS Code Copilot | `.agents/skills/` | Same as standard |
| OpenCode | `.opencode/skills/` | Thin pointer to AGENTS.md or full copy |
| Gemini CLI | `.gemini/skills/` | Thin pointer to AGENTS.md or full copy |

The user selects which providers to generate for during setup. The `.agents/` directory is always generated.

### Skill File Format

Skills follow the [Agent Skills specification](https://agentskills.io/specification):

```markdown
---
name: brainkit-add-brag
description: "Add an accomplishment entry to the bragfile following vault conventions."
user-invocable: true
argument-hint: "[description of accomplishment]"
---

[Skill instructions here...]
```

## Version & Update Awareness

On every run, brainkit:

1. Reads `brainkit.toml` to get the version that last configured this vault.
2. Compares with the currently installed brainkit version.
3. If newer, shows what's new: new features available, updated skill content, etc.
4. Lets the user adopt new features or update existing skills.
5. Writes the new version to `brainkit.toml`.

Additionally, brainkit checks the npm registry for newer versions and shows a non-blocking notice if an update is available.

## Technology Stack

| Component | Technology | Why |
|---|---|---|
| Runtime | Node.js (npm package) | Cross-platform, trivial distribution |
| Language | TypeScript | Type safety, better DX |
| CLI framework | TBD (e.g., Commander, oclif) | Minimal — single command |
| TUI prompts | @clack/prompts or Inquirer | Beautiful interactive prompts |
| Config format | TOML | Clean, human-readable, standard |
| TOML parsing | @iarna/toml or smol-toml | Reliable TOML read/write |
| Skill format | Agent Skills spec (YAML frontmatter + markdown) | Interoperability |
| Template engine | TypeScript builder (no template engine) | AGENTS.md built programmatically, easier to test |
