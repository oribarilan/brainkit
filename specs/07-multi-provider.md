# Multi-Provider Support

## Overview

Brainkit generates agent skill files for multiple AI coding agent providers. This ensures the vault works regardless of which agent the user prefers — Copilot, Claude Code, Cursor, OpenCode, Gemini CLI, etc.

The approach follows the pattern established by [impeccable](https://github.com/pbakaus/impeccable): maintain source skills in a canonical format, then transform/copy them into provider-specific directories.

## Provider Architecture

### Primary: `.agents/` directory

The `.agents/` directory is the emerging cross-provider standard for agent skills. It is always generated and contains the full skill files.

```
.agents/
  skills/
    brainkit/SKILL.md
    brainkit-doctor/SKILL.md
    brainkit-add-brag/SKILL.md
    ...
```

### Secondary: Provider-specific directories

Additional provider directories contain either full copies or thin pointers to the primary `.agents/` skills. The user selects which providers to generate for during setup.

| Provider | Directory | File | Strategy |
|---|---|---|---|
| `.agents/` (standard) | `.agents/skills/<name>/` | `SKILL.md` | Full skills (always) |
| Claude Code | `.claude/skills/<name>/` | `SKILL.md` | Full copy or pointer |
| Cursor | `.cursor/skills/<name>/` | `SKILL.md` | Full copy or pointer |
| OpenCode | `.opencode/skills/<name>/` | `SKILL.md` | Full copy or pointer |
| Gemini CLI | `.gemini/skills/<name>/` | `SKILL.md` | Full copy or pointer |
| Codex CLI | `.codex/skills/<name>/` | `SKILL.md` | Full copy or pointer |
| VS Code Copilot | `.agents/skills/<name>/` | `SKILL.md` | Shares `.agents/` |

### Pointer Strategy

For providers that support it, a thin pointer skill can reference the main AGENTS.md instead of duplicating all skill content:

```markdown
---
name: brainkit
description: "Second brain managed by brainkit — see AGENTS.md for full instructions"
---

This brain is managed by brainkit. See the root `AGENTS.md` file for complete
instructions on how to interact with this vault, including structure, conventions,
writing style, and behavioral rules.

For available commands, see the skills in `.agents/skills/brainkit-*/`.
```

This minimizes file duplication while ensuring each provider discovers the vault instructions.

## Config

Provider selection is stored in `brainkit.toml`:

```toml
[agents]
providers = ["agents", "claude", "cursor"]
```

The `"agents"` provider is always included and cannot be removed.

## Generation Process

When brainkit installs or updates skills:

1. Generate/update all skill files in `.agents/skills/` (the canonical source).
2. For each additional provider in the config:
   - Create the provider's directory structure if needed.
   - Copy or generate the pointer skill file.
3. Clean up: remove skill files for providers that were deselected.

## Adding New Providers

The provider list is maintained in brainkit's source code. Adding a new provider requires:

1. Adding the provider config (directory path, file naming convention).
2. No changes to skill content — all providers use the same SKILL.md format.

This is designed to be a low-effort addition as new AI coding agents emerge.

## AGENTS.md vs. Skills

AGENTS.md and skills serve complementary purposes:

| Aspect | AGENTS.md | Skills (SKILL.md) |
|---|---|---|
| Loaded by | Most agents auto-load AGENTS.md | Agents that support the Agent Skills spec |
| Content | Full vault documentation | Specific actionable commands |
| Invocation | Always active (background context) | User-invoked via slash commands |
| Scope | Everything about the vault | One focused operation per skill |

Both are generated by brainkit. AGENTS.md is the comprehensive reference; skills are focused tools.
