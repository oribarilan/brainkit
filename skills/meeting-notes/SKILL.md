# Meeting Notes

Structured notes from meetings, filed under the relevant PARA directory based on context.

## Location

- Related to an active project → `01_projects/<project-name>/`
- Related to an ongoing area → `02_areas/<area-name>/`
- General or recurring meeting → `03_resources/meetings/` or relevant resource dir
- Use judgment based on what the meeting is about.

## Naming Convention

`YYYY-MM-DD-topic.md` (e.g., `2026-04-03-api-redesign-kickoff.md`)

## Template

```markdown
# Meeting: Topic Name

**Date**: 2026-04-03
**Attendees**: **Sarah Chen**, **Marcus Johnson**, **self**

## Summary

Brief 2-3 sentence summary of what the meeting was about and what was decided.

## Discussion

Key points discussed, organized by topic.

## Decisions

- **Decision**: Description of what was decided

## Action Items

- [ ] **Owner**: Task description — deadline if known
```

## Conventions

- **Bold all people names** — cross-reference with contacts.
- **Bold all decisions and action items.**
- Use `self` to refer to the vault owner.
- Keep it concise — capture decisions and actions, not transcripts.
- Use `brain_write` to create the file in the right location.

## When to Create

Create meeting notes when the user says "I just had a meeting about...", pastes a transcript/agenda, or asks to summarize a discussion. Ask clarifying questions if needed: "Who was in the meeting? What project is this for?"
